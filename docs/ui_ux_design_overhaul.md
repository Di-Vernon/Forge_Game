# 검 만들기 게임 — UI/UX 디자인 대격변 종합 문서

**작성일:** 2026-04-09
**버전:** 1.0

> 이 문서는 게임의 전체 비주얼/UX를 "앱"에서 "게임"으로 전환하는 대격변 설계서이다.
> 3건의 Deep Research 결과 + Jeffrey의 직접 구상을 통합하여 작성.
> Claude Code 프롬프트 제작의 기반 문서로 사용한다.

---

## 1. 디자인 철학

### 핵심 원칙 (5대 원칙)

1. **2~3초 예감 구간이 게임 필의 핵심** — 강화를 즉시 해결하지 않는다
2. **12색 제한 팔레트** (따뜻한 30~40도 hue) — 이것이 "게임" 정체성을 만든다
3. **모든 상호작용에 시각+청각+운동 3중 피드백** — 하나라도 빠지면 "앱"이 된다
4. **성공/실패 에너지 차이 극대화** — 색온도, 음량(10~15dB), 애니메이션 길이 모두 달라야
5. **불완전함이 비밀 재료** — 파티클 +-30px 지터, 타이밍 +-100ms, opacity 100% 금지

### "앱"과 "게임"의 결정적 차이

| 요소 | 앱 UI | 게임 UI |
|------|-------|---------|
| 반응 속도 | 즉시 | 예감-플래시-결과-여운 (4단계) |
| 버튼 | 플랫, 클릭 즉시 반응 | 눌림감(translateY), 그림자 압축, 스프링 복귀 |
| 배경 | 단색 또는 그라디언트 | "장소"로서의 배경 (대장간 내부, 화로, 도구) |
| 텍스트 | 즉시 변경 | 롤링, 페이드, 스케일 바운스 |
| 피드백 | 색상 변경 정도 | 셰이크 + 파티클 + 플래시 + 사운드 동시 |

기존 #0c0906 + #c89028 유지하면서 확장. 다크소울 대장장이 화면과 동일 계열.

## 3. 화면 구조 대격변

### 3-1. 기존 구조 (폐기)

- Home과 Forge가 완전히 다른 화면으로 전환
- 사이드 패널이 항상 고정
- 화면 전환에 연속성 없음

### 3-2. 신규 구조 (Jeffrey 구상)

핵심 개념: 중앙 영역은 항상 유지. 좌우 패널만 슬라이드로 교체.

#### 메인 화면 (Home) 상태

```
+----------+----------------------+----------+
|          |                      |          |
|  보관함   |                      |  칭호    |
|  조합소   |    게임 이름          |  설정    |
|  상점     |    [FORGE 버튼]      |  도감    |
|  도감     |    검 표시            |          |
|          |                      |          |
+----------+----------------------+----------+
```

#### Forge 진입 트랜지션 (스타크래프트식)

1. FORGE 버튼 클릭
2. 좌측 메뉴 패널 -> 왼쪽 화면 밖으로 슬라이드 아웃 (300ms, power2.inOut)
3. 우측 메뉴 패널 -> 오른쪽 화면 밖으로 슬라이드 아웃 (300ms, power2.inOut)
4. 동시에 Forge 전용 패널이 양 옆에서 슬라이드 인:
   - 좌측: 판매 패널 (가격, SELL, 보관)
   - 우측: 강화 패널 (확률, UPGRADE, 비용)
5. 중앙의 검은 제자리에서 자연스럽게 유지

#### Forge -> Home 복귀

역방향 동일 애니메이션 (Forge 패널 아웃 -> Home 메뉴 인)

#### 구현 힌트

```tsx
// 단일 화면, 패널만 교체
<div className="game-container">
  <aside style={{ transform: `translateX(${isForge ? '-100%' : '0'})`,
                  transition: 'transform 300ms cubic-bezier(0.45,0,0.55,1)' }}>
    {isForge ? <ForgeLeftPanel /> : <HomeLeftPanel />}
  </aside>
  <main><ForgeCenter /></main>
  <aside style={{ transform: `translateX(${isForge ? '0' : '100%'})`,
                  transition: 'transform 300ms cubic-bezier(0.45,0,0.55,1)' }}>
    {isForge ? <ForgeRightPanel /> : <HomeRightPanel />}
  </aside>
</div>
```

### 3-3. 상점 화면 (Jeffrey 구상)

컨셉: 플랫 그리드가 아닌, 실제 중세 상점 공간.

- 상품(복원 스크롤 등)이 줄에 매달려 벽에 걸려 있음
- 마치 벽에 걸린 상품처럼, 상자로 이끌 수 있게 걸려있음
- 아래에 카운터 책상 + 물건더미
- 상품 클릭 -> 구매 모달
- 대장간 옆 잡화상에 들어온 느낌

### 3-4. 조합소 화면

상점과 유사한 공간 컨셉:

- 벽에 레시피 재료들이 걸려있음 (조각 종류별로 진열)
- 카운터 위에 조합 도구 (도가니, 망치 등)
- 재료 선택 -> 카운터로 내려옴 -> 조합 버튼 -> 조합 실행 연출

### 3-5. 보관함 화면

- 벽면 무기 랙 - 검들이 세로로 걸려있음
- 클릭 -> 판매/강화 계속 모달 (기존 기능 유지)
- 비어있는 슬롯은 빈 걸이

### 3-6. 파괴 화면

극도로 간소화:

- 검은 화면만 표시. 검 이미지/에셋 일체 없음
- 중앙에 "파 괴" 텍스트 (다크소울 "YOU DIED" 스타일)
  - #8b2500 색상, 75% opacity (100% 금지)
  - 2초간 fade-in, ease-in
  - letter-spacing: 0.5em
- 아래에 파괴된 검 이름 (작게, 뮤트 색상)
- 선택지: 줍기 / 복원 스크롤

## 4. 중앙 영역: 대장간 배경 (5-레이어 아키텍처)

### 레이어 구성

```
Layer 5 (최상): React DOM - UI 텍스트, 검 이름, 레벨
Layer 4:        CSS 파티클 - 떠다니는 불씨 (CSS animation, 10~20개)
Layer 3:        Canvas - 화로 불꽃 파티클 시스템 (rAF, 60~150개)
Layer 2:        픽셀아트 PNG - 대장간 내부 배경 이미지 (image-rendering: pixelated)
Layer 1 (최하): CSS 그라디언트 - 비네트 + 화로 warmglow
```

### Layer 1: CSS 비네트 + 화로 빛

```css
.forge-vignette {
  position: absolute; inset: 0; pointer-events: none;
  background: radial-gradient(
    ellipse 60% 70% at 50% 55%,
    transparent 0%, rgba(12,9,6,0) 30%,
    rgba(12,9,6,0.4) 55%, rgba(12,9,6,0.75) 75%,
    rgba(12,9,6,0.92) 100%
  );
}

.forge-warmglow {
  position: absolute; inset: 0; pointer-events: none;
  background: radial-gradient(
    ellipse 40% 50% at 50% 60%,
    rgba(200,144,40,0.08) 0%, rgba(139,37,0,0.04) 40%, transparent 70%
  );
  mix-blend-mode: screen;
  animation: fire-flicker 3s ease-in-out infinite;
}

@keyframes fire-flicker {
  0%   { opacity: 0.88; }
  10%  { opacity: 0.95; }
  20%  { opacity: 0.90; }
  35%  { opacity: 1.00; }
  50%  { opacity: 0.88; }
  65%  { opacity: 0.93; }
  80%  { opacity: 0.97; }
  100% { opacity: 0.88; }
}
```

### Layer 2: 대장간 배경 PNG

- PixelLab으로 생성할 픽셀아트 배경
- 포함 요소: 돌벽, 화로, 연기, 도구 걸이, 선반, 잉곳 더미
- image-rendering: pixelated
- 중앙에 검이 표시될 공간을 비워둠

### Layer 3: Canvas 불씨 파티클

- 화면 하단 20%에서 스폰
- 색상: #c89028, #e8a030, #ff6a00, #ff4500
- 크기: 1~4px, 상승: 0.3~1.2px/frame, 좌우 +-0.5px drift
- 수명: 60~180프레임 (1~3초), 총 개수: 데스크톱 150캡, 모바일 50
- ctx.shadowBlur=6 + shadowColor 매칭. React state 금지 - useRef+rAF

### Layer 4: CSS 먼지 파티클

- 10~20개 느린 부유 입자, 8~12초 주기, opacity 0.1~0.25

### Layer 5: 검 + UI 텍스트

- 검 PNG (450~520px, image-rendering: pixelated)
- 검 이름 + 레벨: 검 위 중앙
- rotate 없음 - 원본 이미지 그대로 표시

## 6. 피드백 애니메이션 시스템

### 6-1. 4단계 범용 구조

1. 예감 (0~30%): ease-in
2. 플래시 (30~50%): instant
3. 결과 공개 (50~70%): ease-out
4. 여운 (70~100%): spring overshoot

### 6-2. 일반 성공 (300~500ms)

- 0ms: 백색 플래시 (#FFF, opacity 0.8->0, 150ms, mix-blend-mode:screen)
- 50ms: 셰이크 (intensity 8, +-3px, 6단계, 250ms, power2.out)
- 100ms: 불꽃 파티클 폭발 (30~50개, 방사형, 금/주황/빨강/흰)
- 200ms: 레벨 텍스트 바운스 (scale 0->1.2->1.0, back.out(1.7), 500ms)

### 6-3. 일반 실패 (250~400ms)

- 0ms: 빨간 플래시 + 셰이크 (intensity 4)
- 100ms: 화면 어두워짐 (brightness 0.7, saturate 0.5, 300ms)
- 200ms: 실패 텍스트

### 6-4. 파괴 (500~700ms)

- 0ms: 빨간 플래시 + 강한 셰이크 (intensity 12, +-10px, 8단계)
- 100ms: 화면 금(crack) - SVG stroke-dashoffset, 300ms, 50ms 스태거
- 150ms: 화면 완전 어두워짐
- 300ms: "파 괴" 텍스트 - 0->75% opacity, 2s fade-in

### 6-5. Near-miss (기존 유지 + 강화)

- 0ms: 성공인 척
- 300ms: 하드컷
- 500ms: 파괴 시퀀스

### 6-6. 칭호 획득

- 0ms: 화면 어두워짐 (opacity 0->0.7, 300ms)
- 300ms: 칭호명 등장 중앙 (scale 0->1.2->1.0, back.out(1.7))
  - 4층 text-shadow glow
- 1500ms: 설명 fade-in
- 3000ms: 전체 fade-out (500ms)

### 6-7. 마일스톤 (+15, +20, +25)

- 1,500~3,000ms 풀 시네마틱. 탭/클릭 스킵 가능.

### 6-8. 적응형 페이싱

- 기본 400~600ms, 10회 연속 후 200~350ms, 빠른 모드 80~150ms
- 마일스톤은 항상 풀 연출

## 9. 사운드 설계

### 앰비언스 3-레이어

- 기반: 화로 크래클링 + 풀무 바람 (4~8초)
- 중간: 금속 공명 + 굴뚝 바람
- 디테일: 먼 망치 (5~15초) + 석탄 꺼짐 (8~20초)

### 강화 타이밍 동기화

- 0ms: 기계적 쿵 + 금속 울림
- 200~2000ms: 상승 피치 + 모루 타격 가속
- 2000~2500ms: 전략적 침묵 (300~500ms)
- 2500ms+: 성공=차임(1~2kHz) / 실패=둔탁(200~500Hz)

### 피로 방지

- 피치 +-2~3 반음 랜덤, 5~8개 변형, 100ms 재트리거 간격
