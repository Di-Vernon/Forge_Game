# 시뮬레이션 히스토리 & 교훈

## 근본 교훈 (v1~v10에서 반복 검증)

1. **재료 체인은 비재귀적이어야 함.** 재료가 재료를 요구하면 기대비용이 기하급수적으로 폭발. (v6)
2. **floor(n/2) 낙하는 비효과적.** 복귀 과정에서 연쇄 낙하, 76%가 +0~+7에 갇힘. 85%+ 확률 필요 → 게임성 소멸. (v7)
3. **"X% 보호" 모델이 작동.** 달성률 100% 달성. 단, 보호 발동 시 재료 소모 → Phase C 병목. (v8)
4. **건너뛰기로 E_farm→0 가능.** 재료 검을 조각+골드로 즉시 획득하면 Phase C 극적 감소. (v9 방향)
5. **칭호 획득 조건이 너무 쉬우면 Phase A/B가 사라짐.** 불굴 5분, 검성 8분 → 게임 모토 파괴. (v9)
6. **"+N 이상 파괴" 조건은 sell_at ≤ N인 유저를 배제.** 칭호 조건의 사전 공개(UI)로 해결 가능. (v10)
7. **건너뛰기 비용이 높으면 역효과.** 골드 경제 압박 → 완료율 폭락. (v10 Sim-B2)
8. **Phase C 재료 병목의 핵심: 보호 발동 시 재료 소모.** 전 재료의 52~65%가 헛된 소모. (v8, v10)
9. **칭호 전환(잔해↔검성) 전략이 Phase C 자원 문제를 메커니즘으로 해결.** 숫자 조정 대신 플레이어 행동 변화로 해결. (v10 이후 논의)
10. **검성 보호 시 재료 보존이 Phase C 병목의 열쇠.** 보호 발동 시 재료 소모 → 전 재료의 52~65% 허비. 보존으로 변경 시 Phase C 14h → 2.6h. (v11)
11. **해석적 재료 파밍 비용 계산 시 보호율을 반영해야 함.** protect=0으로 계산 시 +16 파밍 = 3,411c (2.84h). protect=0.55 적용 시 256c (0.21h). 재료 파밍은 Phase C 이후 검성 해금 후 주로 발생하므로 protect=0.55가 현실적. (v11)

## 시뮬레이션 설계 원칙

- **해석적 풀이 우선**: Markov chain DP(numpy) → Monte Carlo 검증
- **타임아웃 필수**: 개별 시뮬 max_clicks 제한. E(n) > 수억인 단계는 MC 불가
- **visits[n] 추적**: 각 state 방문 횟수 → 재료 소모량 = visits[n] × count × E_farm
- **병렬 에이전트**: 독립 태스크 병렬, 의존 순차. 자기완결 컨텍스트 + 표준 출력 포맷
- **시간 환산**: 클릭 ÷ 1200 = 시간 (20클릭/분 × 60분)

- 불굴 조건: 15회 파괴 → +8 이상에서 15회 파괴
- 검성 조건: 백야 최초 or 무형검 3회 → 백야(+17) 최초 제작만
- 건너뛰기 비용: 조각 1개 + 골드 대폭
- Phase A: ambitious 기준 25분 ✅, cautious 45시간 ❌ → 교훈 #6
- Phase B: 1~2시간 범위 ✅
- Phase C: 13.7시간 (3배 초과) → 교훈 #7, #8
- **잔해의 수집가 100%+2배 변경 + 칭호 전환 전략 제안** → 교훈 #9

### v11: 검성 재료 보존 + 확률 상향 + 전면 밸런싱

**핵심 변경**:
- 검성의 대장장이 보호 발동 시 재료 검 보존 (v10 최대 병목 해결)

- 잔해의 수집가: 100% 드랍 + 80/20 수량 + 2배 (코드 반영 완료)
- 조합소 레시피 수량 전면 확정 (블러디 쇼텔+16 제외)
- 건너뛰기 골드 비용 확정 (칭호별 특정 조각)

**결과**:

**v10 대비 개선**:
